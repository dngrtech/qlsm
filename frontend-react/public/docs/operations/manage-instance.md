# Manage A Running Server

Use host-level actions and instance-level actions together.

## Host-Level Operations

Use host row **Actions** for:

- restart host
- auto-restart policy
- workshop update push
- QLFilter install/uninstall
- host delete/remove

Reference: [Host Actions Menu](/docs/operations/host-actions-menu)

## Instance-Level Operations

Use instance row **Actions** for:

- edit config
- restart / stop / start
- 99k LAN rate toggle
- logs/chat logs
- RCON console
- delete instance

Reference: [Instance Actions Menu](/docs/operations/instance-actions-menu)

## Config Change Flow

1. Open **Edit Config**.
2. Save changes.
3. Let reconfigure/restart complete.
4. Verify with logs and live behavior.

Instance config edits are isolated to that instance.
